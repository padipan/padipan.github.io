การตั้งค่าเครือข่ายเบื้องต้น

เรียนรู้พื้นฐานการตั้งค่าเครือข่ายคอมพิวเตอร์ที่จำเป็นสำหรับการเชื่อมต่อที่มีประสิทธิภาพ ไม่ว่าจะเป็นการตั้งค่า IP Address, Subnet Mask, Gateway และ DNS

เริ่มเรียนรู้เลย

แนวคิดหลักในการตั้งค่าเครือข่าย

IP Address

ที่อยู่เฉพาะของอุปกรณ์ในเครือข่าย เปรียบเสมือนบ้านเลขที่ที่ใช้ในการระบุตำแหน่งและส่งข้อมูล

Static IP: กำหนดค่าคงที่

Dynamic IP: ได้รับจาก DHCP อัตโนมัติ

ตัวอย่าง: 192.168.1.100

Subnet Mask

ตัวกำหนดขอบเขตของเครือข่าย ช่วยแยกส่วนของ Network ID และ Host ID ในที่อยู่ IP

Class C: 255.255.255.0

Class B: 255.255.0.0

CIDR: /24 = 255.255.255.0

Default Gateway

ประตูเชื่อมต่อไปยังเครือข่ายภายนอก โดยทั่วไปคือที่อยู่ IP ของเราเตอร์

หน้าที่: ส่งข้อมูลออกนอกเครือข่ายท้องถิ่น

ตำแหน่ง: มักเป็น IP แรกหรือสุดท้ายในเครือข่าย

ตัวอย่าง: 192.168.1.1

DNS Server

บริการแปลงชื่อโดเมนเป็นที่อยู่ IP เพื่อให้เข้าถึงเว็บไซต์ได้โดยใช้ชื่อแทนตัวเลข

Google DNS: 8.8.8.8, 8.8.4.4

Cloudflare: 1.1.1.1, 1.0.0.1

หน้าที่: แปลง example.com → 93.184.216.34

คู่มือการตั้งค่าเครือข่ายแบบทีละขั้นตอน

การตั้งค่า IP Address แบบ Static

  1. เปิดการตั้งค่าเครือข่าย

    Windows: คลิกขวาที่ไอคอนเครือข่าย > เปิดการตั้งค่าเครือข่ายและอินเทอร์เน็ต

    macOS: System Preferences > Network

  2. เลือกการเชื่อมต่อเครือข่าย

    เลือกการเชื่อมต่อที่ต้องการตั้งค่า (Ethernet หรือ Wi-Fi)

  3. เปลี่ยนการตั้งค่า IP

    Windows: คลิกที่ Properties > Internet Protocol Version 4

    macOS: คลิกที่ Advanced > TCP/IP

  4. เลือกใช้ IP Address แบบ Static

    เลือก "Use the following IP address" หรือ "Configure IPv4: Manually"

  5. กรอกข้อมูลการตั้งค่า

    กรอก IP Address, Subnet Mask, Default Gateway และ DNS Server

    คลิก OK หรือ Apply เพื่อบันทึกการตั้งค่า

การตรวจสอบการเชื่อมต่อเครือข่าย

คำสั่งพื้นฐานสำหรับตรวจสอบเครือข่าย

1. คำสั่ง ipconfig / ifconfig

ใช้ตรวจสอบการตั้งค่า IP ปัจจุบันของอุปกรณ์

Windows: ipconfig /all

macOS/Linux: ifconfig

2. คำสั่ง ping

ใช้ตรวจสอบการเชื่อมต่อไปยังอุปกรณ์หรือเว็บไซต์

ping 192.168.1.1

ping www.google.com

3. คำสั่ง tracert / traceroute

ใช้ตรวจสอบเส้นทางการเชื่อมต่อไปยังปลายทาง

Windows: tracert www.google.com

macOS/Linux: traceroute www.google.com

ทดสอบความรู้

ลองทำแบบทดสอบสั้นๆ เพื่อประเมินความเข้าใจเกี่ยวกับการตั้งค่าเครือข่ายเบื้องต้น

1. IP Address ใดต่อไปนี้เป็น Private IP Address?

A. 8.8.8.8
B. 192.168.1.100
C. 203.0.113.5
D. 104.16.132.229

คำอธิบาย: 192.168.1.100 เป็น Private IP Address ที่ใช้ในเครือข่ายภายใน ไม่สามารถเข้าถึงได้โดยตรงจากอินเทอร์เน็ต

2. Subnet Mask 255.255.255.0 มีค่าเท่ากับ CIDR ใด?

A. /16
B. /20
C. /24
D. /32

คำอธิบาย: Subnet Mask 255.255.255.0 มีบิต 1 จำนวน 24 บิต จึงเท่ากับ CIDR /24

3. หน้าที่หลักของ DNS Server คืออะไร?

A. จัดสรร IP Address ให้กับอุปกรณ์ในเครือข่าย
B. กำหนดเส้นทางการส่งข้อมูลในเครือข่าย
C. แปลงชื่อโดเมนเป็น IP Address
D. เชื่อมต่อเครือข่ายท้องถิ่นกับอินเทอร์เน็ต

คำอธิบาย: DNS Server ทำหน้าที่แปลงชื่อโดเมน (เช่น www.google.com) เป็น IP Address เพื่อให้คอมพิวเตอร์สามารถเชื่อมต่อได้

4. Default Gateway คืออะไร?

A. อุปกรณ์ที่เชื่อมต่อเครือข่ายท้องถิ่นกับเครือข่ายอื่น
B. เซิร์ฟเวอร์ที่จัดเก็บข้อมูลสำรอง
C. โปรแกรมป้องกันไวรัสและมัลแวร์
D. ระบบจัดการผู้ใช้งานเครือข่าย

คำอธิบาย: Default Gateway คืออุปกรณ์ (โดยทั่วไปคือเราเตอร์) ที่ทำหน้าที่เชื่อมต่อเครือข่ายท้องถิ่นกับเครือข่ายอื่นหรืออินเทอร์เน็ต
